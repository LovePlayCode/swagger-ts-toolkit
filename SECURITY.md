# 安全策略

## 支持的版本

我们为以下版本提供安全更新：

| 版本 | 支持状态 |
| --- | --- |
| 1.x.x | ✅ |

## 报告漏洞

如果你发现了安全漏洞，请不要通过公开的 GitHub issue 报告。

### 报告方式

1. **私下报告**: 通过 GitHub 的私有漏洞报告功能
2. **邮件报告**: 发送详细信息到项目维护者邮箱

### 报告内容

请包含以下信息：

- 漏洞类型
- 受影响的版本
- 重现步骤
- 潜在影响
- 建议的修复方案（如果有）

### 响应时间

- **确认收到**: 48 小时内
- **初步评估**: 7 天内
- **修复发布**: 根据严重程度，14-90 天内

### 安全更新流程

1. 验证和评估漏洞
2. 开发修复方案
3. 内部测试
4. 发布安全更新
5. 公开漏洞详情（在修复发布后）

## 安全最佳实践

### 使用建议

- 始终使用最新版本
- 定期更新依赖
- 在生产环境中谨慎配置远程 URL
- 验证输入的 Swagger 文档来源

### 配置安全

- 不要在配置文件中硬编码敏感信息
- 使用环境变量管理敏感配置
- 限制文件系统访问权限

感谢你帮助保持项目的安全！
